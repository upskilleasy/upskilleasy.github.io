SetUp VM & Install Jenkins

Prerequisite:

Create a VM - AWS EC2, with following configurations:

AMI: Ubuntu v20.04 LTS

Storage volume size: 20GiB

        Instance type: t2.micro (free tier)

        Security Group: Allow ports (22, 80, 8080)

When you select Ubuntu, it will select the latest version 22, in order to select version 20, click on the AMI options dropdown and select the specific version.

Confirm that in the security group, the inbound port 8080 is allowed, otherwise edit it and add the config.

You also need to allow for port 22 (SSH) for connecting the VM from your local machine. Allowing port 80 (HTTP) is also fine.

Example:

Note: For AWS EC2 default user name for the Ubuntu image is ubuntu.

Now, Connect to VM by terminal and execute following steps:

About Advanced Packaging Tool (APT):

The apt command is a powerful command-line tool, which works with Ubuntu. The commands contained within apt provide the means for installing new software packages, upgrading existing software packages, updating the package list index, and even upgrading the entire Ubuntu system.

Imp. apt commands are: install, remove, update and upgrade

  1. IMP: First let’s get the latest updates:

sudo apt update && sudo apt upgrade -y

2. Verify Installed Java Version

java -version

(Java11 recommended)

3. Install Java 11 on Ubuntu Machine (If not installed already)

sudo apt-get -y install openjdk-11-jdk-headless openjdk-11-jre-headless

4. Verify Installed Java Version again

java -version

5. Set JAVA_HOME Variables

  • sudo nano /etc/environment

6. Add the following line at the end of the file and save it.

  • JAVA_HOME=/usr/lib/jvm/java-11-openjdk-amd64

6-a)To save file: Ctrl + O

6-b) When asking for the “File Name to Write: /etc/environment” Press Enter.

6-c) To exit from the editor: Ctrl + X

7. To load the new vars for terminal session (or reboot the vm)

source /etc/environment

8. First, add the repository key to the system

wget -q -O - https://pkg.jenkins.io/debian-stable/jenkins.io.key | sudo apt-key add -

9. Next, append the Debian package repository address to the server’s sources.list

sudo sh -c 'echo deb http://pkg.jenkins.io/debian-stable binary/ > /etc/apt/sources.list.d/jenkins.list'

10. Update System packages

sudo apt update

11. Install Jenkins and its dependencies

sudo apt -y install jenkins

12. Start Jenkins using systemctl

sudo systemctl start jenkins

13. Get Service Status

sudo systemctl status jenkins

14. Enable Jenkins Service (If machine reboots, service will auto start)

sudo systemctl enable jenkins

15. To access jenkins

Open browser and type: <public-ip>:8080

This will prompt for Initial password.

This will be asked for just one time, when accessed for the first time.

16. Use the cat command to display the password:

sudo cat /var/lib/jenkins/secrets/initialAdminPassword

17. Select “Install suggested plugins”

It will start installing the plugins.

18. Create First Admin User

Start Using Jenkins

You will be able to see the Dashboard as below:

19. If you logout and login again, use the user/password you created earlier.

20. Congratulations! Your Jenkins Server is up, running and accessible.
